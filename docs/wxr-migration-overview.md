# baserCMS5 WXR移行プラグイン 企画・仕様メモ

## 目的

- baserCMS5 から WordPress へ移行するために、WXR 形式ファイルを書き出すプラグインを用意する。
- WordPress から baserCMS5 へ移行するために、WXR 形式ファイルを取り込むプラグインを用意する。
- いずれも「WordPress との移行ツール」という位置付けで、管理画面から扱えるようにする。
- 管理画面の体験は BcCsvImportCore を参考にし、ジョブ一覧、処理状況、履歴、エラー確認の流れを共通化する。

## 前提整理

- WXR は WordPress eXtended RSS 形式で、投稿、固定ページ、カテゴリー、タグ、カスタムフィールド、コメント、ユーザー情報などを XML で表現できる。
- baserCMS の全機能が WXR に自然対応するわけではないため、初期版では「WordPress に移しやすい対象」に絞るべき。
- baserCMS 固有機能であるメール、ウィジェットエリア、テーマ設定、コンテンツテンプレート、カスタムコンテンツの複雑なフィールド構造は、初版では対象外または限定対応にする。
- WordPress のメディア移行は WXR 単体では完結しにくく、添付ファイル URL や別途ファイル同期の方針が必要になる。

## 名称方針

### 採用名称

| 用途 | プラグイン名 | 管理画面表示名 | 理由 |
|---|---|---|---|
| baserCMS5 から WXR を出力 | BcWpExport | WordPressエクスポート | 短く覚えやすい。WordPress 連携プラグインとして直感的で、Import と対になる。 |
| WXR から baserCMS5 へ取込 | BcWpImport | WordPressインポート | 用途が分かりやすく、今後 WordPress 連携機能を増やす際にも系列化しやすい。 |

### 代替候補

| 用途 | 候補 | コメント |
|---|---|---|
| エクスポート | BcWxrExport | WXR 専用ツールであることは明確だが、利用者向けにはやや技術寄り。 |
| エクスポート | BcWordPressExport | 直感的だが少し長い。 |
| エクスポート | BcWpMigrationExport | 移行用途は伝わるが、プラグイン名としては少し重い。 |
| インポート | BcWxrImport | WXR 専用であることは明確だが、技術用語が前面に出る。 |
| インポート | BcWordPressImport | 直感的だが長め。 |
| インポート | BcWpMigrationImport | 移行用途は明確だが長い。 |

### 名称方針メモ

- 実装名は BcWpExport / BcWpImport を採用する。
- 管理画面の見出しやメニュー名は、技術用語より利用者向けに「WordPressエクスポート」「WordPressインポート」とする。
- import と export は別プラグインとして分離する。
- 将来的に共通処理を切り出す場合は BcWpCore を追加し、その上に BcWpExport / BcWpImport を乗せる構成が自然。

### 構成方針

- BcWpImport と BcWpExport はそれぞれ独立プラグインとして作成する。
- BcWpImport をメインプラグインにして export 機能を内包する構成は採用しない。
- 理由は、画面構成、ジョブ種別、エラーの扱い、権限、テスト観点が import と export で大きく異なるため。
- 将来共通化したい処理は、実装が見えてから BcWpCore へ切り出す。

## ドキュメント構成

- このドキュメントは共通方針と全体設計の親ドキュメントとして扱う。
- BcWpImport の詳細設計は [docs/plugin-plans/bc-wp-import-design.md](docs/plugin-plans/bc-wp-import-design.md) に分離する。
- BcWpExport の詳細設計は [docs/plugin-plans/bc-wp-export-design.md](docs/plugin-plans/bc-wp-export-design.md) に分離する。
- 当面は親ドキュメントにも要約を残し、実装に近い詳細は子ドキュメントを正とする。

### 共通方針

- 参考 UI は BcCsvImportCore の管理画面とする。
- 画面上部に新規実行フォームを配置する。
- 画面中段に未完了ジョブ一覧を配置し、再開、ダウンロード、削除をまとめる。
- 画面下段に最近の履歴をアコーディオンで配置する。
- 処理中は専用セクションへ切り替え、進捗バー、件数、状態文言を表示する。
- エラーは画面内表示と、詳細ダウンロードの両方を用意する。

### 共通で流用したい要素

- 未完了ジョブ一覧
- 履歴一覧
- 進捗バー
- オプションアコーディオン
- ジョブ削除
- 中断からの再開
- エラー詳細ダウンロード

### エクスポート画面での差分

- CSV ファイル選択の代わりに、出力対象やフィルタ条件の選択フォームを置く。
- 完了ジョブには WXR ダウンロードボタンを表示する。
- エラー CSV の代わりに、警告ログまたは未出力項目レポートのダウンロードを置く。

### インポート画面での差分

- BcCsvImportCore にかなり近い構成で流用しやすい。
- WXR ファイル選択、事前確認、インポート開始、エラー詳細表示を中心にする。
- CSV ではなく XML なので、アップロード直後に「構造解析」と「内容サマリ確認」のステップを追加する。

## 個別プラグイン要約

### BcWpImport

- WordPress の WXR を baserCMS5 へ取り込むプラグイン。
- 初期対象は固定ページ、ブログ記事、カテゴリー、タグ、著者マッピング。
- 解析フェーズと取込フェーズを分けたジョブ管理型にする。
- 詳細設計は [docs/plugin-plans/bc-wp-import-design.md](docs/plugin-plans/bc-wp-import-design.md) を参照する。

### BcWpExport

- baserCMS5 のコンテンツを WordPress 向け WXR として出力するプラグイン。
- 初期対象は固定ページ、ブログ記事、カテゴリー、タグ、投稿者の最小情報。
- 抽出、生成、レポート出力を分けたジョブ管理型にする。
- 詳細設計は [docs/plugin-plans/bc-wp-export-design.md](docs/plugin-plans/bc-wp-export-design.md) を参照する。

## ジョブ設計の共通方針

- import 用と export 用でジョブテーブルは分ける。
- 一時ファイル、ログ、進捗、結果件数をジョブ単位で管理する。
- 詳細なカラム設計と migration たたき台は各子ドキュメントを正とする。

## 実装物の分担

- 共通方針、命名、優先順位はこの親ドキュメントで管理する。
- 画面遷移、ディレクトリ構成、ジョブテーブル、migration は各子ドキュメントで管理する。

## 実装優先順位の提案

### 第1段階

- BcWpImport を先行実装する。
- 理由は、WordPress から baserCMS へ移行したい需要の方が明確で、WXR 入力の解析基盤を先に作ると Export 側にも知見を流用しやすいため。

### 第2段階

- BcWpExport を実装する。
- 固定ページとブログ記事に対象を絞った MVP から始める。

### 第3段階

- BcWpCore の切り出しを検討する。
- コメント、添付ファイル、カスタムコンテンツなどの拡張範囲を見直す。

## 決定しておきたい論点

- 初版でコメントを対象に含めるか。
- メディアファイル本体の移行を初版に含めるか。
- 固定ページの URL 構造をどこまで保持するか。
- WordPress ブロックエディタ由来 HTML をそのまま持つか、整形するか。
- import の重複判定キーを何にするか。

## 現時点の推奨結論

- 名称は BcWpExport / BcWpImport。
- 管理画面表示名は WordPressエクスポート / WordPressインポート。
- import と export は別プラグインとして実装する。
- 参考 UI は BcCsvImportCore の管理画面をベースにする。
- MVP は固定ページとブログ記事に絞る。
- 添付ファイル本体、コメント、カスタム投稿タイプは次フェーズに回す。
